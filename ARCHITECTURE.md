# Architecture

## System Design

This repository is a central backup governance agent for Azure VM backup compliance.
It is designed to run at subscription scope and to replicate policy-like behavior using scripted checks,
remediation planning, and GitHub-native reporting.

### Central Agent Pattern

- **Single source of truth** lives in this repository: `config/backup-rules.yaml`, `config/vault-mapping.yaml`, scripts, and workflows.
- **Subscription-wide scanning** is performed by `scripts/check-backup.ps1`.
- **Actionable remediation plans** are generated by `scripts/generate-plan.ps1` and stored as JSON and Markdown.
- **GitHub workflows** surface findings in run summaries, upload artifacts, and create a PR when a plan is actionable.
- **Remediation is explicit**: this repo generates a plan, then human reviewers approve before any changes are applied.

### Components

1. **Check Module** (`scripts/check-backup.ps1`)
   - Scans Azure VMs and Recovery Services vaults in the subscription
   - Evaluates backup status using `owner` and `environment` tags
   - Produces a structured compliance report in `output/reports/compliance.json`

2. **Plan Module** (`scripts/generate-plan.ps1`)
   - Identifies non-compliant VMs that need backup protection enabled
   - Selects a target vault and policy based on vault mapping and VM location
   - Produces a repeatable remediation plan in `output/plans/remediation.json`
   - Also produces a human-readable plan markdown in `output/plans/remediation.md`

3. **Apply Module** (`scripts/apply-backup.ps1`)
   - Enables backup protection for a VM against a target Recovery Services vault and policy
   - Intended for use after plan review and merge approval

### Configuration

- **Backup Rules**: `config/backup-rules.yaml` defines which environments require backup and which policy names should be used.
- **Vault Mapping**: `config/vault-mapping.yaml` maps subscription/vault targets so the agent can choose a backup vault.

### Workflows

- **Scan workflow** (`.github/workflows/compliance-scan.yml`)
  - Runs subscription-wide or resource-group-scoped scans
  - Writes a GitHub step summary and uploads report artifacts

- **Remediation plan workflow** (`.github/workflows/remediationplan.yml`)
  - Generates an actionable remediation plan
  - Pushes a branch and creates a PR only when there are real backup actions to take
  - Publishes plan metadata and summary in the workflow run

- **Validation workflow** (`.github/workflows/validate.yml`)
  - Verifies PowerShell, YAML, and Bicep quality for changes to agent code and configuration

### Output

- `output/reports/compliance.json`
- `output/plans/remediation.json`
- `output/plans/remediation.md`

### Governance Principles

- No Azure Policy is deployed by this agent. It is a subscription-level governance tool that checks and plans remediation instead of enforcing policy directly.
- Tag-driven decisions ensure that VMs without `owner` or `environment` tags are surfaced for review rather than blindly remediated.
- Human review remains the gate for actual backup enablement.
